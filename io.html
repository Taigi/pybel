Input and Output¶

Parsing, validation, and analysis of of BEL graphs

pybel.from_lines(lines, **kwargs)[source]¶

Loads a BEL graph from an iterable over the lines of a BEL script. This can be a list of strings, file, or other. This function is a very thin wrapper around BELGraph().

Parameters:
  • lines (iter) – an iterable of strings (the lines in a BEL script)
  • kwargs – keyword arguments to pass to BELGraph()
Returns:

a parsed BEL graph
Return type:

BELGraph

pybel.to_pickle(graph, output)[source]¶

Writes this graph to a pickle object with nx.write_gpickle

Cast as a nx.MultiDiGraph before outputting because the pickle serializer can’t handle the PyParsing elements within the BELGraph class.

Parameters:
  • graph (BELGraph) – a BEL graph
  • output – a file or filename to write to

pybel.to_neo4j(graph, neo_graph, context=None)[source]¶

Uploads a BEL graph to Neo4J graph database using py2neo

Parameters:
  • graph (BELGraph) – a BEL Graph
  • neo_graph (py2neo.Graph) – a py2neo graph object, Refer to the py2neo documentation for how to build this object.
  • context (str) – a disease context to allow for multiple disease models in one neo4j instance. Each edge will be assigned an attribute pybel_context with this value
